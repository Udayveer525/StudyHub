# 🎨 StudyHub Frontend — MASTER PLAN

## Core Design Philosophy (Lock This First)

Your frontend must communicate **3 things within 10 seconds**:

1. This platform is **academic & serious**
2. It is **modern and thoughtfully designed**
3. It is **easy to navigate without explanation**

If the evaluator understands it without asking questions → you win.

## 🏠 1️⃣ Landing Page (First Impression Page)

This is the **most important page**.

### Purpose

* Introduce StudyHub
* Show value immediately
* Encourage login/signup

---

### Layout (Top → Bottom)

#### 🔹 Navbar (Always visible)

* Logo: **StudyHub**
* Links:

  * Home
  * Resources
  * Q&A
  * Login / Signup (or Profile if logged in)

Clean, minimal, no clutter.

---

#### 🔹 Hero Section (Above the fold)

Big, bold, confident.

**Headline**

> *A smarter way to prepare for exams.*

**Subtext**

> PYQs, curated resources, and a focused academic Q&A — all in one place.

**Primary CTA**

* “Get Started” (Signup)
* Secondary: “Browse Resources” (No login)

Visual:

* Abstract illustration (books, code, graph, checklist)

#### 🔹 Feature Highlights (3–4 cards)

* 📚 **Exam-focused resources**
* 🧠 **Structured Q&A (not a chat)**
* ⭐ **Community-validated answers**

Short, scannable.

#### 🔹 Footer

## 🔐 2️⃣ Authentication Flow (Simple, Clean)

### Pages

* `/login`
* `/signup`

### Signup

Minimal fields:

* Name
* Email
* Password

Optional note:

> *More personalization available in dashboard.*

Do NOT ask for:

* Phone
* University
* Semester
* Degree

Those go in dashboard later.

---

### Login

* Email
* Password
* Remember me (optional visual, but JWT handles this)

### UX Notes

* Clear error messages
* No popups
* Full page forms (not modals)

---

## 📚 4️⃣ Resources Section (Core Academic Value)

### Page: `/resources`

#### Step-by-step flow:

1. Select Degree
2. Select Semester
3. Select Subject
4. See:

   * PYQs
   * Videos
   * Notes (later)
   * Books (later)

### UI Style

* Left side: filters
* Right side: content cards

Each resource card:

* Title
* Year
* Type badge (PYQ / VIDEO)
* Open link button

No downloads — only links.

---

## ❓ 5️⃣ Q&A Section (Professional, Not Chatty)

### Page: `/questions`

Layout:

* Left: filters (degree, semester, subject)
* Center: question list

Each question card:

* Title
* Subject tag
* Answer count
* Status badge (Open / Answered)

---

### Ask Question

Button:

> “Ask a Question”

Form:

* Subject (mandatory)
* Title
* Description (optional)

No markdown editor initially — plain textarea is fine.

---

### Question Detail Page

* Question at top
* Answers below
* Accepted answer highlighted
* Vote buttons (+ / –)
* Answer form at bottom (auth only)

This page should feel **serious and focused**.

---

## 🧠 Tech Stack (Frontend)

Recommended:

* React
* Vite
* Tailwind CSS
* React Router
* Axios / Fetch

This is:

* Fast to build
* Modern
* Very presentable

---

## 🔒 Auth Handling (Frontend Logic)

* Token stored in `localStorage`
* On app load:

  * Call `/auth/me`
  * If success → logged in
  * Else → logout

Protected routes:

* Dashboard
* Ask Question
* Answer submission
